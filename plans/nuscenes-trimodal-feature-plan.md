# 计划：nuScenes 三模态场景特征提取与投影

> 来源 PRD：`PRD.md`（nuScenes三模态场景特征提取与投影）

## 架构性决策

以下决策在所有阶段中保持稳定：

- **脚本边界**：固定为两脚本架构：`extract_features.py`（提取）与 `project_and_visualize.py`（投影与可视化），二者可独立运行。
- **数据顺序契约**：150 个验证集 scene 的顺序仅在提取脚本中确定一次，并作为所有输出文件的行对齐基准。
- **数据形状契约**：特征矩阵固定为 `features_agent_raw.npy (150,17)`、`features_map_raw.npy (150,8)`、`features_ego_raw.npy (150,9)`。
- **元数据契约**：`metadata.parquet` 与三份特征逐行对齐，包含 `scene_token`、`scene_name`、`location`、`log_name`，并预留未来指标列。
- **归一化策略**：按模态独立执行 `RobustScaler`（median + IQR），原始特征必须保留落盘。
- **参数扫描策略**：UMAP 固定 `random_state=42`，扫描 `n_neighbors ∈ {5,10,20,30}` 与 `min_dist ∈ {0.0,0.1,0.5}`。
- **验证口径**：MVP 以可视化肉眼验证为准，不引入聚类指标。

## 阶段 1：流水线骨架与顺序基线

**覆盖用户故事**：1, 15, 19

### 要建设内容

打通最小可运行的端到端链路：初始化 nuScenes、按确定性顺序遍历全部 150 个验证集 scene、写出基础元数据，并确保提取与投影脚本可独立执行。若任一 scene 提取失败，流程应明确报错并终止。

### 验收标准

- [ ] 可一次性遍历全部 150 个验证集 scene，且顺序稳定可复现。
- [ ] `metadata.parquet` 成功生成，且行数与 scene 数一致。
- [ ] 提取脚本与投影脚本可独立运行，不相互强耦合。
- [ ] 任一 scene 提取失败时，输出清晰错误并中止流程。

## 阶段 2：Agent 模态垂直切片

**覆盖用户故事**：2, 3, 4, 5, 6, 7

### 要建设内容

完成 Agent 17 维特征的端到端链路：40 帧统计均值、50m 半径过滤、动态/静态分母分离、距离分段密度、速度统计 NaN 安全处理、v1 严重遮挡占比，并落盘为原始特征矩阵，保证后续可直接进入投影脚本。

### 验收标准

- [ ] Agent 特征矩阵输出形状为 `(150, 17)`。
- [ ] 动态类别占比、距离分段密度、速度统计、遮挡比例均可稳定产出。
- [ ] 速度相关统计对单帧轨迹导致的 NaN 具备鲁棒性（不报错且结果有效）。
- [ ] 在同一 scene 顺序下与元数据逐行对齐。

## 阶段 3：Map 模态垂直切片

**覆盖用户故事**：8, 9, 10

### 要建设内容

完成 Map 8 维特征的端到端链路：在关键帧索引 `[0, 9, 19, 29, 39]` 进行采样，以 ego 为圆心 30m 圆形区域提取道路拓扑与区域占比特征，并对 5 次采样结果求均值后落盘，支持直接投影。

### 验收标准

- [ ] Map 特征矩阵输出形状为 `(150, 8)`。
- [ ] 车道、连接线、道路分段、路口存在性与区域占比特征完整可用。
- [ ] 面积占比类特征值范围合理（0 到 1）。
- [ ] 5 帧采样均值逻辑可复现且与元数据逐行对齐。

## 阶段 4：Ego 模态垂直切片

**覆盖用户故事**：11, 12, 13

### 要建设内容

完成 Ego 9 维特征的端到端链路：仅依赖 pose 差分构建速度/加速度/转向相关统计，计算低速过滤后的累计曲率与低速帧占比，不依赖 CAN bus，覆盖全部验证集场景并落盘后可直接投影。

### 验收标准

- [ ] Ego 特征矩阵输出形状为 `(150, 9)`。
- [ ] 直行场景曲率接近 0、转弯场景曲率显著大于 0（方向正确）。
- [ ] 低速帧占比定义稳定、可复现，且不依赖停走事件检测。
- [ ] 全流程不依赖 CAN bus 文件，避免覆盖缺口。

## 阶段 5：落盘契约、预留列与归一化闭环

**覆盖用户故事**：14, 16, 20

### 要建设内容

固化数据契约：三模态原始特征必须落盘；投影阶段按模态独立执行 `RobustScaler`；`metadata.parquet` 预留未来模型指标列，确保后续叠加性能信息无需迁移既有 schema。

### 验收标准

- [ ] 三份 raw 特征文件稳定输出且与元数据同序对齐。
- [ ] 归一化仅在投影脚本中进行，不覆盖 raw 文件。
- [ ] `metadata.parquet` 包含必需字段与预留指标字段。
- [ ] 任意单模态可被单独加载、缩放并进入投影。

## 阶段 6：UMAP 参数扫描、出图矩阵与可视化校验

**覆盖用户故事**：17, 18

### 要建设内容

为三模态分别执行 UMAP 4×3 参数网格扫描并保存共 36 张散点图，统一按 `location`（Boston/Singapore）着色；将 PRD 中约定的可视化合理性信号作为上线前校验门槛。

### 验收标准

- [ ] 每个模态产出 12 张图，共计 36 张 PNG，命名与参数一一对应。
- [ ] 所有散点图均按 `location` 着色，无需重新初始化 nuScenes。
- [ ] UMAP 随机性固定（`random_state=42`），重复运行结果可对比。
- [ ] 若所有参数组合均未出现预期分离信号，流程会触发特征构建复核而非直接进入后续扩展。
